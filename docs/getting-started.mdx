# Memulai dengan Humanika

Panduan lengkap untuk mengatur dan menjalankan sistem Humanika di environment development Anda.

## üìã Prasyarat Sistem

Sebelum memulai, pastikan sistem Anda memenuhi persyaratan berikut:

- **Node.js**: Versi 18.0.0 atau lebih tinggi
- **npm**: Versi 8.0.0 atau lebih tinggi
- **MongoDB**: Database MongoDB (lokal atau cloud)
- **Git**: Untuk cloning repository

### Verifikasi Prasyarat

```bash
# Cek versi Node.js
node --version

# Cek versi npm
npm --version

# Cek versi Git
git --version
```

## üöÄ Instalasi

### 1. Clone Repository

```bash
git clone <repository-url>
cd humanika
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Setup Environment Variables

Buat file `.env.local` di root directory project:

```env
# Database
DATABASE_URL="mongodb://localhost:27017/humanika"

# NextAuth Configuration
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your-nextauth-secret-here"

# Google OAuth (untuk authentication)
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"

# Google Drive API (untuk file management)
GOOGLE_DRIVE_API_KEY="your-google-drive-api-key"
```

### 4. Setup Database

```bash
# Generate Prisma client
npm run prisma:generate

# Push schema ke database
npm run prisma:push

# Seed database dengan data awal
npm run prisma:seed
```

### 5. Jalankan Development Server

```bash
npm run dev
```

Aplikasi akan berjalan di `http://localhost:3000`

## üîß Konfigurasi Google Services

### Google Cloud Console Setup

1. **Buat Project Baru**

   - Kunjungi [Google Cloud Console](https://console.cloud.google.com/)
   - Buat project baru atau pilih project existing

2. **Enable APIs**

   - Google Drive API
   - Google OAuth 2.0 API

3. **Buat OAuth 2.0 Credentials**

   - Application type: Web application
   - Authorized redirect URIs: `http://localhost:3000/api/auth/callback/google`

4. **Dapatkan API Key**
   - Untuk Google Drive API access

## üóÑÔ∏è Database Schema

Humanika menggunakan Prisma ORM dengan MongoDB. Schema database mencakup:

- **Users**: Manajemen pengguna dan roles
- **Periods**: Periode kepengurusan organisasi
- **Management**: Jabatan dan struktur organisasi
- **WorkPrograms**: Program kerja departemen
- **Events**: Kegiatan dan acara
- **Documents**: Manajemen dokumen
- **Finance**: Transaksi keuangan
- **Articles**: Konten artikel
- **Galleries**: Galeri foto

### Inisialisasi Data

Setelah setup database, jalankan seeding untuk data awal:

```bash
npm run db:seed
```

## üêõ Troubleshooting

### Error: "MongoDB connection failed"

**Solusi**: Pastikan MongoDB berjalan dan DATABASE_URL benar

```bash
# Cek MongoDB connection
mongosh --eval "db.adminCommand('ismaster')"
```

### Error: "Google OAuth not working"

**Solusi**: Verifikasi credentials Google Cloud Console

1. Pastikan redirect URI benar
2. Pastikan OAuth consent screen dikonfigurasi
3. Pastikan APIs diaktifkan

### Error: "Prisma client not generated"

**Solusi**: Regenerate Prisma client

```bash
npx prisma generate
```
